## 2026-07-03 — Step 9: Hybrid Recommendation Engine implemented

- 2 new migrations: `user_interactions` (`user_id`/`product_id` nullable, `interaction_type`, `weight`, `metadata` JSON, `created_at` only), `recommendation_logs` (`user_id`/`product_id`, `module`, `algorithm_source`, `score`, `confidence`, `reason`, `shown_at`/`clicked_at`). Deliberately **not** built: `user_preferences`, `trending_scores` — see PROJECT_DECISIONS.md for the caching-instead-of-precomputed-tables reasoning.
- New `InteractionType` backed enum (9 cases: Viewed, Wishlisted, WishlistRemoved, CartAdded, CartRemoved, Purchased, Rated, RecommendationClicked, SearchQuery) with per-type default weights (removal types are negative signals) — `config/recommendation.php` can override any weight without touching code.
- New `app/Repositories/RecommendationRepository.php` — sole data-access point for both new tables; every algorithm service goes through it, never queries `user_interactions`/`recommendation_logs` directly.
- New `app/Services/Recommendation/` namespace: `InteractionTrackingService` (9 `record*()` methods, thin wrapper over the repository), `ContentBasedService` (weighted category/color/price affinity profile + item-item similarity for "Similar Products"), `CollaborativeFilteringService` (Jaccard similarity over interaction sets, no ML/training), `PopularityService` (blended trending/viewed/wishlisted/rating/new-arrival signal, always computable — the universal fallback), `HybridRecommendationService` (confidence-weighted blend with proportional weight redistribution when an algorithm returns nothing), `RecommendationService` (the only class controllers depend on — `forYou()`/`similarProducts()`/`trending()`/`algorithmOnly()`/`logShown()`/`recordClick()`, wraps every call with execution-time + cache-hit logging), `RecommendationCacheService` (version-counter cache keys, since `CACHE_STORE=database` doesn't support `Cache::tags()`), `RecommendationEvaluator` (leave-one-out Precision@K/Recall@K/MAP@K/NDCG@K/Coverage/Diversity/Novelty).
- New DTOs: `RecommendationScore` (content/collaborative/popularity/final scores, confidence, reason, algorithm source, timestamp) and `RecommendationResult` (product + score, `reason()` helper) — final, readonly classes; Blade templates only ever touch `product` and `reason()`.
- New Artisan command: `php artisan recommendations:evaluate --algorithm=all --k=10` — prints a Precision/Recall/MAP/NDCG/Coverage/Diversity/Novelty comparison table across content/collaborative/popularity/hybrid.
- New `recommendations` log channel (`config/logging.php`, daily rotation, 14-day retention) — every recommendation-generation event logs algorithm, user, result count, execution time, and cache hit/miss.
- New `RecommendationController` (`recommendations.index`, `recommendations.click`) + routes: `GET /recommendations` (replaces the `/coming-soon` fallback the nav's "Recommendations" link used since Step 4) and `GET /recommendations/click/{product}` (tracking redirect — logs the click, marks the originating log row, redirects to the real product page).
- Interaction tracking wired into existing controllers: `ProductCatalogueController::show` (records a view + logs "Similar Products" shown), `SearchController::index` (records the search query), `WishlistController::store`/`destroy` (records wishlist add/remove + invalidates that user's cache), `CartController::store` (records cart addition + invalidates cache), `Vendor\ProductController::update`/`destroy` (bumps the global cache version — a product edit or deletion can affect any user's recommendations).
- New reusable components: `<x-recommendation-card>` (product-card visual language + a visible `reason` line + a click-tracking href) and `<x-recommendation-section>` (title + grid + empty-state, used identically on Home, Product Detail, and the dedicated Recommendations page). New stylesheet: `resources/css/components/recommendations.css`.

- Product detail page: "Similar Products" section (content-based item-item similarity, 6 items) below the existing product info.
- New `/recommendations` page: full grid (24 items), an algorithm switcher (`?algorithm=hybrid|content|collaborative|popularity`) for side-by-side academic comparison, empty state when an algorithm has no signal for the current user.
- **4 real bugs found and fixed during this step's own verification** (cache-key collision missing `$limit`, `RecommendationLog` timestamp misconfiguration, leave-one-out evaluator leaving a stray interaction row behind, click-redirect using the product's ID instead of its slug) — full detail in PROJECT_DECISIONS.md.
